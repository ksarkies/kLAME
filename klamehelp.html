kLAME is intended for conversion of audio wav to mp3 files using the LAME audio file conversion tool. It provides a convenient GUI for LAME, allowing a number of conversions to be made in bulk for different LAME options (e.g. archival compression, plus compression for transmission or website storage).

(Top) Main Form

The main window provides a table in which each row represents a wav file for conversion, and each column represents a conversion option set that includes LAME options, a destination directory and a filename tag to be added to the end of the base filename, and used to describe the converted files.

The window provides a toolbar with a set of buttons described below. Columns are provided with checkboxes that can be used to control if the filename will be converted according to the options set for the particular column. If unchecked, no conversion will take place for the corresponding file and options set.

Open/Save Project. The full set of columns, options and LAME options can be saved for later restoration. When saved, the file is given an extension of .prj. Note that the filenames are not saved.

Open Files. This dialogue allows the user to select a list of filenames and have them added to a table for later conversion. Additional selections will result in the additional files being added.

Delete Row. Clicking will remove the selected row.

Add Columns. When pressed an additional column is added with default options. The column will be added at the end of the existing columns and will be given a number following that of the last column.

Delete Column. Clicking will remove the selected column and all associated settings.

LAME Options. When a new column is added, LAME options will normally need to be set. These will be set for the selected column, or if no column is selected, for the last column. LAME options are described separately below.

Convert. Begin the conversion operation. This performs all conversions requested, silently, and informs with a message box when complete. At present there is no way to interrupt this.

(Top) LAME Options

This tabbed form provides for any option to be passed to LAME. Some of the more common options can be set by form controls and fields, while the less common options can be added in an edit field in the same format as that used by the command-line LAME tool. For the meanings of the LAME options, consult the documentation associated with LAME. The most accurate source appears to be that provided by running lame - -longhelp and lame - -preset help.

(Top) Tab 1: General Output options:

Column number. This refers to the column for which options are being set. You can edit the column header to something a bit more readable.

Output directory. Allows selection of a different output directory for each column.

Filename Tag. Allows for a tag to be added to the end of the base filename (before the first dot) to distinguish the different conversion products.

Save LAME settings. This saves a simple one-line “options string” identical to that passed to LAME as options. These saved options can be used for creating standard option settings.

Load LAME settings. When a saved settings file is loaded, the options string from that file is parsed and the settings are made into the options form.

(Top) Tab 2: Basic LAME Settings

Preference. This chooses between VBR (quality) and ABR/CBR (bitrate).

Relative Quality. This is a slider between 10 and 100, and provides the VBR quality setting associated with -v. The LAME option associated with -v is determined by subtracting the slider value from 100 and dividing by 10.

VBR Algorithm. Chooses between --vbr-old and --vbr-new.

Bitrate and "use CBR". If CBR is checked, this sets --cbr and -b n. If not checked this sets --abr n. The slider provides the value of n.

(Top) Tab 3: Advanced LAME settings

Mode (-m) Various mode settings.

Quality (-q). Setting from a spin box 0-9 with default 5.

Full Bandwidth (-k). No filtering applied.

High Pass Frequency (kHz). Filters for restricting the frequencies of the signal before conversion.

High Pass Width (kHz).

Low Pass Frequency (kHz).

Low Pass Width (kHz).

Tonality Limit (kHz). Limit to which audible frequencies are processed.

Use Minimum VBR Bitrate (-b). If checked, a VBR minimum bitrate of 115kHz is used. This can cause problems with LAME if the value is too low compared to the average bitrate.

Use Maximum VBR Bitrate (-B). If checked, a maximum bitrate can be set on the slider. This can also cause problems with LAME.

Resample Frequency. If checked, the selected resample frequency (allowed values given in the spinbox) is used. This setting produces a set of samples of the signal at a different rate than that of the original. It can be used to reduce the final bitrate (and so file size) where quality is of less importance.

Change Quality Measure (-X). Measures 0 to 9 given in a spinbox. This changes the way that quality is measured by LAME during conversion.

Additional options. This allows any other options to be specified. These are parsed and, if the checkbox is selected (default), invalid options are discarded. Note that the invalid options set may change as LAME is developed. Any options added that clash with those set on the form will take precedence. Any LAME option can be specified. Options relating to terminal outputs are ignored.

(Top) Project Save file structure

This is a binary file (QT format) with an ASCII header containing the program name and date, followed by:

  1. Number of columns

  2. Directory for the wav files

  3. Comment strings

  4. Filename tags

  5. Output directories

  6. LAME option strings

  7. Column Header labels
